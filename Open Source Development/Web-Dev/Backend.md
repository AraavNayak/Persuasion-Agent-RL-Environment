Choice of language depends on project requirements, team expertise, performance needs.
    - Javascript often used for full stack dev. Used on both frontend and backend.
    - Python/ruby used for rapid development and fast development cycles.
    - Java/C# used for lage scale enterprise grade applications. Used in large orgs and government systems
    - Go used for performance critical apps

Databases
- Relational. data stored in tables in rows and columns that are related. Can use primary and foreign keys to connect tables via relationships (eg. MySQL, PostgreSQL, SQL server, Oracle)
        - Good for apps with complex queries like banking systems, ecommerce apps
        - With all those relational databasees, we retrive and store data via SQL (Structured Query Language)
- Non relational. Good for data without a predefined table structure. Suitable for applications which require flexible data models. (eg. MongoDB, CauchDB, Cassandra)


Web framework
- Dependent on programming language. For Python, Django. For Java, Spring boot. For Javascript, Express.js. For C#, ASP.NET core. For Ruby, Ruby on Rails. For Go, gin.
        - All of these frameworks provide tools for building backends. handling tasks like routing, requests, response handling, database handling
        - Web frameworks allow us to make API's. which allow frontend and backend to 'communicate' (eg. frontend can retreive data from backend)
